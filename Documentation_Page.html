Introduction

CSS is a cornerstone technology used for web development, enabling the separation of content and presentation. It provides a powerful mechanism for controlling the visual aspects of web pages, including layout, colors, fonts, and animations.

Syntax

CSS rules consist of a selector and a declaration block enclosed in curly braces. Selectors target HTML elements, and declaration blocks contain one or more declarations separated by semicolons. Each declaration consists of a property and a value, separated by a colon.

Selectors
  • Element Selector: Targets HTML elements by their tag name. For example, `p` selects all p tag elements.p{ font-family:arial; }
  • Class Selector: Targets elements with a specific class attribute. Syntax: .classname.
  • ID Selector: Targets a single element with a specific ID attribute. Syntax: #idname.
  • Attribute Selector: Targets elements with a specific attribute or attribute value. Syntax: [attribute=value].
  • Pseudo-classes and Pseudo-elements: Selectors that define a special state of an element or parts of an element, such as :hover, :nth-child(), or ::before.
Properties
  • Color: Defines the text color using keywords, hexadecimal, RGB, or HSL values.
  • Font: Sets the font family, size, style (italic, bold), and weight (light, normal, bold).
  • Background: Sets the background color or image for an element.
  • Border: Sets the border properties including color, width, and style (solid, dashed, dotted).
  • Margin: Sets the margin space around an element, controlling its positioning relative to other elements.
  • Padding: Sets the padding space around an element's content, maintaining distance from the border.
  • Width and Height: Sets the dimensions of an element, either fixed values or percentages of the parent container.
  • Positioning: Controls the position of an element within its containing element, offering options like static, relative, absolute, and fixed.
  • Display: Defines how an element is displayed in the layout flow, such as block, inline, inline-block, or none.
Box Model
  • Content: The actual content of the box, where text and images appear.
  • Padding: The transparent area around the content, inside the border.
  • Border: The border surrounding the padding.
  • Margin: The transparent area outside the border, separating the element from other elements on the page.
Specificity

Determines which CSS rule takes precedence when multiple rules target the same element.

Inline styles have the highest specificity, followed by IDs, classes, and element selectors.

Specificity is calculated based on the number of ID selectors, class selectors, and element selectors in a rule.

Media Queries

Used to apply different styles based on the characteristics of the device, such as screen size, orientation, and resolution.

Allows for responsive web design, adapting layouts and styles for various devices and screen sizes.

Transitions and Animations
  • Transitions: Define smooth changes in property values over a specified duration.
  • Animations: Define keyframes for specifying styles at various points during the animation, offering more complex and dynamic effects.
Flexbox and Grid
  • Flexbox: A layout model providing a more efficient way to distribute space among items in a container, making it easier to create flexible and responsive layouts.
  • Grid: A two-dimensional layout system for designing web page layouts in rows and columns, offering precise control over element positioning and alignment.
Units
  • Absolute Units: Pixels (px), points (pt), inches (in), centimeters (cm). These units provide fixed sizes irrespective of other factors.
  • Relative Units: Percentages (%), em, rem, vw (viewport width), vh (viewport height). Relative units adapt to the surrounding context, making them useful for responsive design.
Comments

CSS comments can be added using `/* comment here */`, allowing developers to annotate their code for clarity or temporarily disable styles during testing or debugging.

Integration

CSS can be included in an HTML document via the style tag, linked externally using the link tag, or added inline within HTML elements using the style attribute. External CSS files are commonly used for better organization and maintainability.

Browser Compatibility

While CSS is standardized, certain properties and selectors may behave differently across different web browsers. Testing and vendor-prefixes may be necessary for ensuring consistent rendering across various platforms and browser versions. Cross-browser testing tools can help identify and resolve compatibility issues.
